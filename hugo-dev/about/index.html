Hugo 开发教程

assets目录包含通常通过资产管道传递的全局资源。其中包括图像、CSS、Sass、JavaScript和TypeScript等资源。查看详细信息。

archetypes目录包含新内容的模板。查看详细信息。

config目录包含您的站点配置，可能会拆分为多个子目录和文件。对于具有最小配置的项目或不需要在不同环境中有不同行为的项目，在项目根目录中有一个名为hugo.toml的配置文件就足够了。查看详细信息。

content 目录包含标记文件（通常是Markdown）和组成网站内容的页面资源。查看详细信息。

data 目录包含数据文件（JSON、TOML、YAML或XML），这些数据文件用于增强内容、配置、本地化和导航。查看详细信息。

i18n 目录包含多语言网站的翻译表。查看详细信息。

layouts 目录包含将内容、数据和资源转换为完整网站的模板。查看详细信息。

public目录包含运行hugo或hugo服务器命令时生成的已发布网站。Hugo会根据需要重新创建此目录及其内容。查看详细信息。

resources 目录包含来自Hugo资产管道的缓存输出，这些输出是在运行hugo或hugo server命令时生成的。默认情况下，此缓存目录包括CSS和图像。Hugo会根据需要重新创建此目录及其内容。

静态目录包含在生成站点时将复制到公共目录的文件。例如：favicon.ico、robots.txt和验证网站所有权的文件。在引入页面包和资产管道之前，静态目录也用于图像、CSS和JavaScript。

themes目录包含一个或多个主题，每个主题都在自己的目录中。

union 文件系统

Hugo创建了一个 union 文件系统，允许您将两个或多个目录挂载到同一位置。例如，假设您的主目录在一个目录中包含Hugo项目，在另一个目录中包含共享内容：

home/
└── user/
    ├── my-site/            
    │   ├── content/
    │   │   ├── books/
    │   │   │   ├── _index.md
    │   │   │   ├── book-1.md
    │   │   │   └── book-2.md
    │   │   └── _index.md
    │   ├── themes/
    │   │   └── my-theme/
    │   └── hugo.toml
    └── shared-content/     
        └── films/
            ├── _index.md
            ├── film-1.md
            └── film-2.md

您可以在使用挂载构建站点时包含共享内容。在您的站点配置中：

hugo.toml

[module]
  [[module.mounts]]
    source = 'content'
    target = 'content'
  [[module.mounts]]
    source = '/home/user/shared-content'
    target = 'content'

当您将一个目录覆盖在另一个目录之上时，您必须装载这两个目录。

Hugo 不遵循象征性的联系。如果您需要符号链接提供的功能，请改用 Hugo 的联合文件系统。

挂载后，union文件系统的结构如下：

home/
└── user/
    └── my-site/
        ├── content/
        │   ├── books/
        │   │   ├── _index.md
        │   │   ├── book-1.md
        │   │   └── book-2.md
        │   ├── films/
        │   │   ├── _index.md
        │   │   ├── film-1.md
        │   │   └── film-2.md
        │   └── _index.md
        ├── themes/
        │   └── my-theme/
        └── hugo.toml

当两个或多个文件具有相同的路径时，优先级顺序遵循装载顺序。例如，如果共享内容目录包含books/book-1.md，则将忽略该目录，因为项目的内容目录是先装入的。

您可以将目录挂载到 archetypes， assets， content， data， i18n， layouts 和 static 查看详细信息。

您还可以使用Hugo Modules从Git存储库挂载目录。查看详细信息。
