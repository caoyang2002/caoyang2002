Content

创建模板之前要理解的最重要的概念是上下文，即传递到每个模板的数据。数据可以是简单的值，或更常见的是对象和关联的方法。

例如，单个页面的模板接收Page对象， Page对象提供返回值或执行操作的方法。

在模板中，点 ( . ) 表示当前上下文。

<h2>{{ .Title }}</h2>

在上面的例子中，点代表Page对象，我们调用它的Title方法来返回前面定义的标题。

当前上下文可能会在模板内发生变化。例如，在模板的顶部，上下文可能是Page对象，但我们将上下文重新绑定到range内或块with的另一个值或对象。

<h2>{{ .Title }}</h2>

{{ range slice "foo" "bar" }}
  <p>{{ . }}</p>
{{ end }}

{{ with "baz" }}
  <p>{{ . }}</p>
{{ end }}

在上面的示例中，当我们range值切片时，上下文会发生变化。在第一次迭代中，上下文是“foo”，在第二次迭代中，上下文是“bar”。在with块内部，上下文是“baz”。 Hugo 将以上内容呈现为：

在range内或with块中，您可以通过在点前添加美元符号 ( $ ) 来访问传递到模板中的上下文：

{{ with "foo" }}
  <p>{{ $.Title }} - {{ . }}</p>
{{ end }}

Hugo 将其呈现为：

<p>My Page Title - foo</p>

Action

在上面的示例中，成对的左大括号和右大括号表示模板操作、模板内的数据评估或控制结构的开始和结束。

模板操作可以包含文字值（ boolean 、 string 、 integer和float ）、变量、函数和方法。

{{ $convertToLower := true }}
{{ if $convertToLower }}
  <h2>{{ strings.ToLower .Title }}</h2>
{{ end }}

在上面的例子中：

  • $convertToLower是一个变量
  • true是一个布尔值
  • strings.ToLower是一个将所有字符转换为小写的函数
  • Title是Page对象上的一个方法

Hugo 将以上内容呈现为：

  
  
    <h2>my page title</h2>

空白

注意到前面示例中的空行和缩进了吗？尽管在通常缩小输出时与生产无关，但您可以通过使用带有连字符的模板操作分隔符来删除相邻的空格：

{{- $convertToLower := true -}}
{{- if $convertToLower -}}
  <h2>{{ strings.ToLower .Title }}</h2>
{{- end -}}

Hugo 将其呈现为：

<h2>my page title</h2>

空白包括空格、水平制表符、回车符和换行符。

Pipes

在模板操作中，您可以通过管道将值传递给函数或方法。管道值成为函数或方法的最终参数。例如，这些是等效的：

{{ strings.ToLower "Hugo" }} → hugo
{{ "Hugo" | strings.ToLower }} → hugo

您可以将一个函数或方法的结果通过管道传输到另一个函数或方法中。例如，这些是等效的：

{{ strings.TrimSuffix "o" (strings.ToLower "Hugo") }} → hug
{{ "Hugo" | strings.ToLower | strings.TrimSuffix "o" }} → hug

这些也是等效的：

{{ mul 6 (add 2 5) }} → 42
{{ 5 | add 2 | mul 6 }} → 42

分行

您可以将模板操作拆分为两行或多行。例如，这些是等效的：

{{ $v := or $arg1 $arg2 }}

{{ $v := or 
  $arg1
  $arg2
}}

您还可以将原始字符串文字拆分为两行或多行。例如，这些是等效的：

{{ $msg := "This is line one.\nThis is line two." }}

{{ $msg := `This is line one.
This is line two.`
}}

变量

变量是用户定义的标识符，前面带有美元符号 ( $ )，表示任何数据类型的值，在模板操作中初始化或分配。例如， $foo和$bar是变量。

变量可以包含标量、切片、映射或对象。

使用:=初始化变量，使用=为先前已初始化的变量赋值。例如：

{{ $total := 3 }}
{{ range slice 7 11 21 }}
  {{ $total = add $total . }}
{{ end }}
{{ $total }} → 42

在if 、 range或with块内初始化的变量的作用域为该块。在这些块之外初始化的变量的范围仅限于模板。

对于表示切片或映射的变量，使用index函数返回所需的值。

{{ $slice := slice "foo" "bar" "baz" }}
{{ index $slice 2 }} → baz

{{ $map := dict "a" "foo" "b" "bar" "c" "baz" }}
{{ index $map "c" }} → baz

切片和数组是从零开始的；元素 0 是第一个元素。

使用表示映射或对象的变量、链标识符来返回所需的值或访问所需的方法。

{{ $map := dict "a" "foo" "b" "bar" "c" "baz" }}
{{ $map.c }} → baz

{{ $homePage := .Site.Home }}
{{ $homePage.Title }} → My Homepage

如上所示，对象和方法名称都是大写的。尽管不是必需的，但为了避免混淆，我们建议以小写字母或下划线开头变量和映射键名称。

函数

在模板操作中使用的函数采用一个或多个参数并返回一个值。与方法不同，函数不与对象关联。

Go 的 text/template 和 html/template 包提供了一小组通用的函数、运算符和语句。有关详细信息，请参阅函数文档的go-templates部分。

Hugo 提供了数百个按名称空间分类的自定义函数。例如， strings命名空间包括以下函数和其他函数：

Function 功能 Alias 别名
strings.ToLower lower 小写
strings.ToUpper upper 大写
strings.Replace replace 替换

如上所示，常用的函数都有一个别名。在模板中使用别名可以减少代码长度。

调用函数时，用空格将参数与函数分隔开，参数之间也用空格分隔。例如：

{{ $total := add 1 2 3 4 }}

方法

在模板操作中使用并与对象关联的方法采用零个或多个参数，并返回一个值或执行操作。

最常访问的对象是 Page 和 Site 对象。这是每个对象可用的方法的一小部分样本。

Object 目的 Method 方法 描述
Page Date 返回给定页面的日期。
Page Params 返回给定页面前面定义的自定义参数的映射。
Page Title 返回给定页面的标题。
Site Data 返回由数据目录中的文件组成的数据结构。
Site Params 返回站点配置中定义的自定义参数的映射。
Site Title 返回站点配置中定义的标题。

使用点 ( . ) 将方法链接到其对象，如下所示，请记住前导点代表当前上下文。

{{ .Site.Title }} → My Site Title
{{ .Page.Title }} → My Page Title

有些方法需要一个参数。用空格将参数与方法分隔开。例如：

{{ $page := .Page.GetPage "/books/les-miserables" }}
{{ $page.Title }} → Les Misérables

注释

要尝试使用 HTML 注释分隔符来注释掉模板代码。

Hugo 在渲染页面时去除 HTML 注释，但首先评估 HTML 注释分隔符内的任何模板代码。根据 HTML 注释分隔符中的模板代码，这可能会导致意外结果或构建失败。

模板注释与模板操作类似。成对的左大括号和右大括号代表注释的开始和结束。例如：

{{/* This is an inline comment. */}}
{{- /* This is an inline comment with adjacent whitespace removed. */ -}}

注释中的代码不会被解析、执行或显示。注释可以是内联的，如上所示，也可以是块形式：

{{/*
This is a block comment.
*/}}

{{- /*
This is a block comment with
adjacent whitespace removed.
*/ -}}

您不得将一条评论嵌套在另一条评论中。

要呈现 HTML 注释，请通过safeHTML模板函数传递一个字符串。例如：

{{ "<!-- I am an HTML comment. -->" | safeHTML }}
{{ printf "<!-- This is the %s site. -->" .Site.Title | safeHTML }}

包含

使用template函数包含一个或多个 Hugo 的嵌入模板：

{{ template "_internal/google_analytics.html" . }}
{{ template "_internal/opengraph" . }}
{{ template "_internal/pagination.html" . }}
{{ template "_internal/schema.html" . }}
{{ template "_internal/twitter_cards.html" . }}

使用partial或partialCached函数来包含一个或多个部分模板：

{{ partial "breadcrumbs.html" . }}
{{ partialCached "css.html" . }}

在 layouts/partials 目录中创建部分模板。
